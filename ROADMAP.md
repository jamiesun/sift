# 开源项目审计器 项目画像与方向

> 本文档是北极星和护栏：说清"做成什么样 / 绝不做什么"，把"具体怎么写"留给执行 Agent。
> 项目名：**sift**（CLI 即 `sift`）。语言：Rust。

## 项目概述

一个**可控成本**的开源项目审计工具。在引入一个开源库之前，不必直接试用、也不必让前沿大模型生吞数万行代码，就能拿到一份定位到文件/行号的风险账本，据此决定是否引入。

核心思路是 **分级漏斗 + 算力错配**：脏活累活（结构提取、粗筛压缩）交给零成本静态解析和廉价小模型，高强度逻辑收敛只在最后一步交给前沿大模型。整个工具编译为单一二进制，零环境配置即可对**整个项目**或**单个模块**做审计。

- 架构图

```text
  CLI 参数 / ENV / config.toml
        │  (降级寻址, 取不到 Key 即退出)
        ▼
  扫描层  walkdir + ignore  →  路径流(有界 channel)
        │  (黑名单/.gitignore 在迭代器层切断)
        ▼
  零阶  tree-sitter 解析 → 脱水(签名/import/调用) → 扁平 JSON → drop AST
        │  跨目录引用打 [EXTERNAL_BLACKBOX] 断点
        ▼
  一阶 Map  小模型并发粗筛 → {职责,依赖,风险} 解析失败回退 AST 签名
        ▼
  二阶 Reduce  前沿大模型 · ReACT 状态机 · 技能=本地函数 · 三次熔断
        ▼
  stdout: Markdown 风险清单(含调用链/行号)
```

## 项目画像（目标状态）

- **零摩擦冷启动。** `audit ./repo` 直接能跑。缺配置不打断、不进交互式追问；信息走"参数 > 环境变量 > 配置文件"降级链，全断则立刻退出并给一行如何注入的提示。
- **成本可控、可预算。** 大部分 token 花在廉价小模型上，前沿大模型只接收脱水后的架构骨架。用户能预判"审一个百兆库大概花多少钱"，而不是被一把梭进前沿模型的账单吓退。
- **内存平稳，吞得下大库。** 单进程常驻内存与项目规模脱钩；文件流式处理、处理完即丢，不驻留整棵树。
- **报表能直接做决策。** 输出是定位到文件/行号的风险点（并发死锁、资源耗尽、抽象泄露、可疑外部调用），不是赏析体车轱辘话。半成品也比卡死强。
- **项目审计与模块审计同源。** 二者只是扫描根的切片范围不同，不是两套逻辑；模块审计不会膨胀成全局扫描。
- **单文件分发。** 产物尽量是一个可执行文件，丢到机器上就能跑，无需装运行时或拉依赖目录。
- **品质冲突时的优先级：** 鲁棒不崩 > 报表可用 > 成本低 > 速度快 > 体积小。宁可输出带熔断标记的半成品，也不要进程 panic 或挂死。

## 当前能力清单

- 暂无（greenfield，空仓库）。本文件先于代码存在，作为开发指北。

## 非目标（铁律）

- **不做向量库 / Embedding / RAG 切片。** 单次低频审计下，索引维护成本大于直接拼 prompt，纯文本管道阅后即焚。
- **不做运行时插件 / 动态技能注册。** 技能即编译期写死的本地函数，靠枚举 + match 路由；扩展靠改代码重编译，不靠加载外部模块。
- **不引入向交互界面/服务化漂移。** 它是一次性命令行工具，不是常驻服务、不带 Web UI、不做多租户。
- **不允许主进程因脏数据 panic。** 解析不出的宏/残缺节点静默丢弃记日志；模型幻觉/坏 JSON 触发熔断，绝不放任死循环或崩溃。
- **不靠"直接试用"替代审计。** 价值就在引入前给判断，事后再审失去意义。
- **模块审计不得越界成全局扫描。** 跨目录引用一律打断点交大模型脑补，不盲目追链。

## 方向与意图

- **多语言覆盖。** tree-sitter 语法可插拔，逐步支持主流语言（Rust/Go/Python/JS/TS…），服务"任意开源库都能审"的画像。
- **小模型可替换。** 本地 8B 或廉价 API 皆可接，让用户按预算选粗筛引擎，服务"成本可控"。
- **可机读输出。** 除 Markdown 外可选 JSON，便于接 CI 或二次加工，服务"决策"画像。
- **断点缝合增强。** `[EXTERNAL_BLACKBOX]` 标记将来可选做有限跨目录补全，但默认收敛在指定边界内，守住"模块审计不膨胀"红线。

## 完成的样子

> 当下列结果同时出现，才算这个工具真正可用。

- 对空配置直接运行能跑通，缺 Key 时 0.1s 内退出并给注入提示；不挂起。
- 审一个百兆级库，常驻内存稳定在低位、不随规模线性膨胀；扫坏数据不崩。
- 拿到的报表能定位到文件/行号，含跨模块依赖与至少一类并发/资源风险，开发者据此能直接拍板引不引入。
- 模型连续出错时输出带熔断标记的半成品，而非死循环或 panic。
- 同一二进制既能审整个项目，也能 `--module path/` 审子目录，结果不串。

> 以下是**建议而非铁律**，执行 Agent 可按实际取舍：单二进制、并发解析、降级寻址、有界通道、熔断计数都属画像要求，但具体用 Rust/Go/Python、是否上 rayon、体积/耗时阈值、unwrap 数量等都是手段，按团队顺手与基准结果定，别当验收红线锁死。
